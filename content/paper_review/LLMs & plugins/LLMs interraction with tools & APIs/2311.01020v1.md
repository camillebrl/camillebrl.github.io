Résumé du papier sur les préoccupations des développeurs lors de l'utilisation de GitHub Copilot

Avec le récent progrès de l'intelligence artificielle (IA) et l'émergence de Grands Modèles Linguistiques (GLMs), les outils de génération de code basés sur l'IA ont fait des progrès significatifs et sont devenus une solution pratique pour le développement de logiciels. GitHub Copilot, conçu comme un programmeur associé basé sur l'IA, utilise des modèles d'apprentissage automatique formés sur un grand corpus d'extraits de code pour générer des suggestions de code ou auto-compléter du code en utilisant le traitement du langage naturel.

Malgré sa popularité, il y a peu de preuves empiriques sur les expériences réelles des développeurs de logiciels qui travaillent avec Copilot. Pour combler ce manque, une étude empirique a été menée pour comprendre les problèmes et les défis auxquels les développeurs sont confrontés lorsqu'ils utilisent Copilot en pratique, ainsi que leurs causes sous-jacentes et les solutions potentielles. Les données ont été collectées auprès de 476 problèmes GitHub, 706 discussions GitHub et 184 publications Stack Overflow.

Les résultats révèlent que les problèmes d'_utilisation_ et de _compatibilité_ sont les plus courants chez les utilisateurs de Copilot. Les causes les plus fréquentes de ces problèmes sont les _problèmes internes de Copilot_, les _problèmes de connexion réseau_ et les _problèmes de compatibilité avec l'éditeur/IDE_. Les solutions les plus courantes à ces problèmes sont la _correction des bugs par Copilot_, le _changement de configuration/paramètres_, et l'_utilisation de la version appropriée_.

Ces résultats indiquent des défis significatifs que les utilisateurs doivent surmonter lors de l'utilisation de Copilot, et suggèrent des moyens de l'améliorer pour une meilleure expérience utilisateur.

Les problèmes d'utilisation sont principalement causés par des problèmes internes à Copilot ou une mauvaise connexion réseau. Les problèmes de compatibilité se produisent généralement en raison de problèmes de compatibilité avec l'éditeur/IDE utilisé par le développeur. Les problèmes liés au contenu des suggestions sont généralement dus à des suggestions de basse qualité ou dénuées de sens.

Quatre catégories d'améliorations pour Copilot ont été suggérées. Tout d'abord, il faut améliorer la compatibilité à travers divers environnements IDE et simplifier la configuration de Copilot. Deuxièmement, les utilisateurs ont besoin de plus d'options de personnalisation pour adapter le comportement de Copilot à leur propre flux de travail. Troisièmement, les utilisateurs ont besoin de plus de moyens pour contrôler le contenu généré par Copilot. Enfin, il est nécessaire d'améliorer la qualité du code généré par Copilot.
