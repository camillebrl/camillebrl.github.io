Ruoxi Sun, Sercan O. Arik, Rajarishi Sinha, Hootan Nakhost, Hanjun Dai, Pengcheng Yin, et Tomas Pfister de l'équipe de recherche sur l'intelligence artificielle dans le cloud de Google ont publié un article intitulé "SQLPrompt : In-Context Text-to-SQL with Minimal Labeled Data". Cet article présente "SQLPrompt", une nouvelle approche pour l'apprentissage en contexte avec peu de données labélisées pour la génération de requêtes SQL à partir de textes en langage naturel.

SQLPrompt est particulièrement adapté pour améliorer les capacités des grands modèles de langage à générer des requêtes SQL dans le cadre d'un apprentissage en contexte. Les méthodes utilisées comprennent une conception innovante de sollicitations, une stratégie de décodage basée sur la cohérence d'exécution qui choisit le SQL le plus cohérent parmi d'autres propositions de SQL, et une méthode qui vise à améliorer les performances en diversifiant les propositions de SQL lors de la sélection de la cohérence avec différentes conceptions de sollicitations ("MixPrompt") et des modèles de base différents ("MixLLMs").

Les auteurs montrent que SQLPrompt surpasse les approches précédentes pour l'apprentissage en contexte avec peu de données labélisées avec une grande marge, réduisant l'écart avec l'état de l'art de l'ajustement fin avec des milliers de données labélisées.

Concernant la question de la configuration du problème pour Text-to-SQL, ils notent que : L'ensemble des données de base \(D_{q}\) qui convertissent les requêtes en langage naturel \(q\) en SQL comprend le schéma de la base de données \(S\), les clés primaires \(K_{p}\) et les clés étrangères \(K_{f}\). \(S\) contient généralement plusieurs tables \(T_{t}\), chaque table \(T_{t}\) a un nom \(N_{t}\), des noms de colonnes \(c_{j}\) et des types de données de colonnes \(t_{j}\). Les clés primaires \(K_{p}\) identifient de manière unique les lignes de chaque table, et les clés étrangères \(K_{f}\) rejoignent plusieurs tables.

De plus, les chercheurs ont proposé de nouvelles conceptions de sollicitations pour améliorer la diversité des propositions SQL : les "prompts concis" et les "prompts verbeux". Les "prompts concis" présentent les informations d'une manière linéaire, tandis que les "prompts verbeux" décrivent les bases de données avec des mots compréhensibles par l'homme et mettent l'accent sur les informations que les modèles de langage à grande échelle doivent connaître.

Pour augmenter la diversité des propositions SQL dans le décodage de la cohérence, les chercheurs ont élargi leur méthode pour ne pas utiliser un seul LLM, mais plutôt un mélange de LLMs, en combinant les résultats de différents prompts et de différents LLMs.

Cependant, cette méthode présente une limitation dans le fait que l'interrogation de plusieurs conceptions de prompts et LLMs peut être coûteuse et prendre du temps. Bien que l'idée de combiner plusieurs conceptions de prompts et de LLMs soit prometteuse pour améliorer les performances, il est nécessaire d'explorer des moyens plus efficaces et rentables pour les combiner à l'avenir.
