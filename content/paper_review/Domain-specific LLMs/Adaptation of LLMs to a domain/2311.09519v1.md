Résumé du papier [Leveraging Code to Improve In-context Learning for Semantic Parsing](http://arxiv.org/pdf/2311.09519v1).

L'apprentissage en contexte (ICL) est une approche attrayante pour le parsing sémantique en raison de sa nature en quelques exemples et de sa meilleure généralisation. Cependant, apprendre à analyser des langages spécifiques à un domaine (DSLs) rares à partir de quelques démonstrations est un défi, ce qui limite les performances des modèles de langage à grande échelle (LLMs). Dans leur travail, les auteurs améliorent l'efficacité de l'ICL pour le parsing sémantique en (1) utilisant des langages de programmation généraux comme Python au lieu de DSLs, et (2) en augmentant les prompts avec une description structurée du domaine qui inclut, par exemple, les classes et fonctions disponibles.

Leur approche montre des améliorations significatives de la précision à travers trois ensembles de données populaires. De plus, ils constatent que la ressemblance du langage d'analyse cible avec le code général est un facteur plus important que la popularité du langage dans les corpus de pré-entraînement.

Introduction

Le parsing sémantique, qui est la traduction d'énoncés en langage naturel en représentations structurées de leur signification, est une exigence clé pour la construction de systèmes de dialogue orientés tâches et d'assistants vocaux. Cette tâche est principalement résolue soit en affinant des modèles sur des ensembles de données étiquetés d'énoncés mappés vers des programmes DSL, soit en utilisant l'ICL pour alimenter un LLM avec quelques exemples.

Cependant, chaque stratégie présente des limites significatives. L'affinement nécessite des ensembles importants de données étiquetées, qui peuvent être coûteux et longs à obtenir. De plus, les modèles affinés ont du mal à généraliser de manière compositionnelle, par exemple pour déchiffrer des programmes plus longs que ceux rencontrés pendant l'entraînement ou pour produire des structures jamais vues. L'ICL peut améliorer la généralisation compositionnelle dans certains cas, mais l'apprentissage à partir de quelques démonstrations est un défi.

Dans ce travail, les auteurs explorent deux questions principales: comment utiliser les compétences en codage des LLM pour améliorer le parsing sémantique basé sur l'ICL et si les LLM peuvent mieux généraliser de manière compositionnelle avec des langages de programmation généralistes (PLs) qu'avec des DSLs.

Définitions du domaine

Un résultat important de cette étude est la proposition d'ajouter une description du domaine (DD en anglais) pour aider le modèle à comprendre les fonctionnalités spécifiques au domaine. Les DDs pour PLs sont plus faciles à définir car ils peuvent déclarer explicitement les objets, les méthodes, leurs signatures, etc.

Résultats expérimentaux

Dans un cadre en "quelques exemples", l'approche de l'étude montre que les prompts PL avec DDs améliorent considérablement la précision de l'exécution sur toutes les données testées par rapport à l'approche ICL standard d'un prompt DSL sans DD. De plus, l'utilisation d'un prompt Python avec DD peut souvent éliminer le besoin de nombreuses démonstrations.

Il est important de mentionner que même si la performance de Python est due à sa prévalence dans le corpus de pré-entraînement, l'expérience avec d'autres PLs indique que la prévalence dans les corpus de pré-entraînement n'explique pas nécessairement une performance supérieure.

En conclusion, l'étude démontre que l'utilisation de PLs au lieu de DSLs améliore considérablement l'ICL pour le parsing sémantique et que la fourniture d'une description de domaine est souvent plus efficace que des démonstrations supplémentaires. Les résultats suggèrent une méthode améliorée pour construire des parseurs sémantiques dans le contexte moderne de l'ICL avec les LLM.
