Résumé du papier [Evaluating In-Context Learning of Libraries for Code Generation](http://arxiv.org/pdf/2311.09635v1).

Le document s'intéresse à l'apprentissage en contexte des bibliothèques de code par les modèles de langage à grande échelle (LLM). Les auteurs examinent la capacité des LLM à apprendre de nouvelles bibliothèques de programmation et des langages de programmation moins répandus en contexte, sans formation supplémentaire. Ils ont développé un cadre expérimental pour évaluer la capacité des modèles à générer du code en utilisant des fonctions de bibliothèques inconnues spécifiées dans le contexte.

Le document évalue les modèles sur trois scénarios : (1) dans lequel un modèle doit apprendre une nouvelle bibliothèque pour effectuer une tâche, (2) où le modèle est contraint d'utiliser une bibliothèque donnée pour effectuer une tâche de génération de code général, et (3) où le modèle doit générer du code dans un langage de programmation relativement rare. De plus, les auteurs examinent l'effet de différents types de supervision de contexte, y compris les descriptions en langage naturel ou les implémentations de code, et le rôle sémantique du nom de fonction dans l'apprentissage de bibliothèques de programmation.

Les conclusions sont les suivantes :
- Les LLM peuvent apprendre et utiliser de nouvelles bibliothèques de programmation, et cette capacité n'est pas limitée aux modèles propriétaires les plus grands.
- Les LLM peuvent apprendre de nouvelles bibliothèques à partir de descriptions en langage naturel ou d'implémentations de code aussi efficacement qu'ils peuvent utiliser les démonstrations.
- Les LLM ont du mal à générer du code en utilisant les fonctions de bibliothèque définies en contexte et sont plus performants lorsqu'ils génèrent du code librement sans contraintes.
- Les LLM, comme GPT-4, montrent une capacité non négligeable à apprendre en contexte un nouveau langage de programmation à partir de zéro.

Les auteurs concluent que leurs résultats illustrent le potentiel des LLM pour apprendre et adapter des bibliothèques de programmation en contexte, ce qui peut aider à la personnalisation rapide des LLM pour différentes applications sans nécessiter une nouvelle formation ou un affinage.
