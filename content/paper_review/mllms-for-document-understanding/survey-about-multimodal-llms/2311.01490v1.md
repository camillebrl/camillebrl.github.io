Dans le document [The Behavior of Large Language Models When Prompted to Generate Code Explanations](http://arxiv.org/pdf/2311.01490v1), les auteurs explorent de manière systématique comment les grands modèles de langage (LMMS) génèrent des explications de code à partir d'exemples de type introductif à la programmation. Ils montrent que la nature des explications générées par les LMMS varie considérablement en fonction de plusieurs paramètres. Cependant, ils restent cohérents dans deux aspects majeurs : le degré de lisibilité, qui tourne autour d'un niveau de 7-8e année, et la densité lexicale, qui représente la taille relative des mots significatifs par rapport à la taille totale de l'explication.

Les explications générées sont généralement correctes, mais elles ne sont pas toujours complètes, concises et contextualisées. Les auteurs soulignent également les défis d'utilisation des LMMS, tels que le comportement non déterministe et la génération de "faits" non véridiques. Ils suggèrent donc une approche prudente dans l'utilisation de ces outils sans des études systématiques solides qui peuvent documenter leurs forces et faiblesses.

Le document explore également comment les LMMS ont été utilisés jusqu'à présent pour générer du code à partir de descriptions en langage naturel et pour générer des explications de code. Cependant, il souligne que les travaux antérieurs sur l'utilisation des LMMS pour générer des explications de code sont limités à plusieurs égards. Par exemple, ils n'ont pas toujours rendu compte des invitations précises qu'ils ont utilisées, ils n'ont pas toujours rendu compte d'autres paramètres importants, ou n'ont pas varié ces paramètres de manière systématique.

Pour explorer plus systématiquement les LMMS, les auteurs ont varié cinq paramètres d'entrée majeurs : la formulation de l'invitation, le type d'exemple de code, la température, le modèle LMMS et le langage de programmation. Ils ont généré des explications à partir de 13 invitations, trois LMMS, cinq niveaux de température, cinq exemples de code et trois langages de programmation, pour un total de 3 510 explications de code.

La complexité et le type de code expliqué influent sur la nature des explications générées. L'ajout d'un seul mot à l'invite peut également entraîner des différences importantes dans les explications générées. Les auteurs concluent que les travaux futurs devraient se concentrer sur la compréhension détaillée de la génération et de l'utilisation des explications générées par les LMMS. Ils recommandent également la création d'un référentiel d'invites avec des évaluations des utilisateurs et des paramètres comme la température pour faciliter l'utilisation des LMMS pour la génération d'explications de code.
