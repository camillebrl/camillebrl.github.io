Résumé du papier "Beyond Text: Unveiling Multimodal Proficiency of Large Language Models with MultiAPI Benchmark".

Les modèles de langage à grande échelle, tels que ChatGPT, ont grandement amélioré la compréhension et la génération de langage, avec un impact sur un large spectre d'applications. Cependant, ces modèles excellent principalement dans les tâches basées sur le texte et négligent la complexité des informations multimodales dans le monde réel. Pour résoudre ce problème, cette étude introduit MultiAPI, un ensemble de données de référence innovant et complet visant à étendre la compétence des modèles de langage à grande échelle (LLM) dans des contextes multimodaux.

MultiAPI, développé en collaboration avec ChatGPT, comprend 235 appels d'API diversifiés et 2 038 invitations contextuelles, offrant une plateforme unique pour évaluer les LLM augmentés d'outils pour gérer les tâches multimodales. Les expériences réalisées révèlent que si les LLM démontrent une compétence dans la prise de décision d'appel d'API, ils rencontrent des défis dans l'identification du domaine, la sélection de la fonction et la génération d'arguments. Il a été observé que le contexte auxiliaire peut nuire à la performance.

L'étude propose une approche pour résoudre ces défis en introduisant une description détaillée du domaine et une correction des arguments. L'application de cette approche a montré une amélioration de la performance dans toutes les métriques d'évaluation.

En somme, la recherche introduit un ensemble de données de référence novateur pour les tâches multimodales, évalue de manière exhaustive les LLM et suggère des solutions pour améliorer ces modèles. Elle ouvre la voie à de futures recherches sur les LLM dans le contexte multimodal.

## 1. Introduction

Les modèles de langage à grande échelle (LLM), tels que ChatGPT, ont eu un impact significatif sur de nombreuses tâches du domaine ouvert et ont marqué l'industrie et le monde universitaire. Cependant, ils sont souvent confinés aux domaines et aux tâches basés sur le texte sur lesquels ils ont été formés, négligeant la nature multimodale et dynamique de l'information du monde réel.

Pour adresser ces problématiques, les recherches récentes ont suivi deux approches principales pour intégrer les capacités de traitement multimodal aux LLM existants : 1) L'entraînement conjoint ou le fine-tuning des LLM avec des composants pour le codage et la génération multimodaux. 2) L'introduction d'outils d'API auxiliaires via des interfaces de langage naturel, positionnant les LLM comme l'entité centrale de prise de décision déterminant les outils appropriés à employer pour l'enquête.

Cependant, malgré le potentiel significatif et la flexibilité que les LLM augmentés d'outils expriment sur les tâches multimodales, leur performance quantitative reste insuffisamment examinée. Pour combler ce vide, les chercheurs ont développé MultiAPI, un ensemble de données d'évaluation à grande échelle qui comprend 235 appels d'API fonctionnels et 2 038 instructions.

## 2. Travaux connexes

La performance des LLM a été évaluée de manière approfondie depuis l'introduction de ChatGPT, fournissant des informations précieuses pour améliorer les futures itérations du modèle et aider l'industrie à développer des applications plus résilientes. Cependant, leur compétence à utiliser les outils n'a pas été explorée en profondeur.

D'autres études ont examiné la combinaison de grands modèles de langage avec des ressources externes. Cependant, cette intégration a du mal à produire un code précis en raison d'incohérences dans le code source véridique et de la documentation insuffisante. 

## 3. Ensemble de données de référence MultiAPI

### Collecte des données

MultiAPI est construit en utilisant le jeu de données HuggingFace, à partir duquel les chercheurs ont extrait des modèles de haute qualité dans 9 domaines, avec leurs instructions. Les modèles ont été encapsulés en tant que fonctions d'API en utilisant les invites de ChatGPT, suivies d'affinements humains minutieux pour garantir l'exécution et une argumentation cohérente à travers les domaines.

### Métriques d'évaluation

L'étude introduit un cadre d'évaluation à quatre niveaux pour l'évaluation des LLM. Ce cadre comprend : 1) Évaluation des appels, 2) Correspondance de domaine, 3) Correspondance de fonction, et 4) Correspondance d'argument. 

## 4. Expériences

L'étude a conduit des expériences sur les LLM basés sur API et les LLM open-source. Les résultats révèlent que bien que les modèles prennent avec précision la décision d'invoquer des fonctions d'API, ils ont souvent du mal à sélectionner la bonne fonction et les bons paramètres du bon domaine.

## 5. Analyse

Une analyse d'erreur détaillée a été menée pour comprendre la cause potentielle de tels erreurs, ce qui a conduit à proposer deux solutions simples mais efficaces pour atténuer ces erreurs. Les résultats expérimentaux valident l'efficacité de leur méthode.

## 6. Cadre d'amélioration

Les chercheurs ont testé deux solutions pour résoudre les défis identifiés : l'une consiste à ajouter une description de domaine à l'invite du système du modèle, et l'autre consiste à utiliser un second LLM comme éditeur d'argument pour vérifier et réviser les prédictions d'argument.

## 7. Conclusion

L'étude a introduit un ensemble de données de référence innovant pour les tâches multimodales et a fourni une analyse détaillée des performances des LLM sur l'exécution de tâches multimodales et sur la manière dont les améliorations peuvent être apportées. Les résultats devraient aider à orienter les futures recherches sur l'utilisation des LLM dans les tâches multimodales.
